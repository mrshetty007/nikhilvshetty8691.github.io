2.2 EXCEPTION DASHBOARD

Project Overview

The 2.2 Exception Dashboard is designed to track the return of assets. Once a new asset is delivered, the old asset specified in the request must be returned within 10 days. If the old asset is not returned within this period, a 2.2 exception request is created to track the overdue asset. This dashboard helps visualize all data related to these exceptions, similar to the 2.1 Exception Dashboard.

Key Responsibilities

. Oversee the return of assets after new deliveries.

. Track escalated requests for overdue asset returns. .

. Reduce manual effort in preparing "Ticket creation data."

. Monitor month-wise progress to understand data trends and improve processes .

Files and Tools Involved

. Power BI

. Excel

. AI Tool(Chat GPT)

. Share point to store all the data

. Main Data base is service now

Process

.Collect data from ServiceNow regarding delivered assets from the previous two weeks and prepare the 2 series exception report.

.Extract data for 2.2 exceptions, focusing on assets not returned within the specified timeframe.

.Upload the file to SharePoint and connect it to Power BI through a web connection.

.Prepare the dashboard according to the requirements .

.Create the dashboard according to requirements, including metrics such as total request count, requests closed, requests in progress, escalated requests, escalated percentage, ongoing user responses, follow-up trend charts, escalated trends, month-wise progress, country-wise counts through map charts, and table charts for client call data extraction.

Result

The 2.2 Exception Dashboard significantly improved the tracking and management of overdue asset returns. Automation through Power BI reduced manual efforts and errors, enhancing data accuracy and reliability. The dashboard provided real-time insights and comprehensive visualizations, leading to better exception handling and faster resolution of overdue asset issues. This streamlined operations, saved time, and contributed to improved decision-making processes within the organization.
