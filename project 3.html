2.1 EXCEPTION DASHBOARD

Project Overview

The 2.1 Exception Dashboard is designed to monitor asset activity on the network, specifically tracking whether users or employees have connected to the client network. If an asset is not on the network, the team creates a request in the user's name and sends follow-ups to track the asset's activity. This dashboard helps us manage and track all these requests, providing insights into the progress of 2.1 exceptions. Key metrics displayed include the number of requests closed, total requests, requests in progress, and escalated requests if the user has not connected to the network. Additionally, the dashboard includes various visualizations as shown in image to provide a comprehensive view of the request status and exception handling progress.

Key Responsibilities

. To overview the asset activity after the delivery .

. To keep track of the escalated request .

. To get the manual efforts less in the preparation of "Ticket creation data ." .

. To track the month wise progress and to understand the data and improve the process .

Result

The 2.1 Exception Dashboard significantly improved the monitoring and management of network asset activity. It automated the tracking of requests, reducing manual effort and errors. This led to enhanced data accuracy and reliability. The dashboard provided real-time insights and comprehensive visualizations, enabling better exception handling and faster issue resolution. Overall, it streamlined operations, saved time, and contributed to improved decision-making processes within the organization.
